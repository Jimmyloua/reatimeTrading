  Create authentication controller and DTOs:

  **DTOs (package: com.tradingplatform.auth.dto):**
  - RegisterRequest: email (String, @Email @NotBlank), password (String, @Size(min=8))
  - LoginRequest: email (String, @Email @NotBlank), password (String, @NotBlank)
  - LoginResponse: accessToken (String), refreshToken (String), userId (Long), expiresIn (long - seconds until access token expires)
  - LogoutRequest: refreshToken (String, @NotBlank)
  - RefreshTokenRequest: refreshToken (String, @NotBlank)

  **AuthController.java (package: com.tradingplatform.auth):**
  - @RestController, @RequestMapping("/api/auth"), @RequiredArgsConstructor
  - Inject UserService, JwtTokenProvider, AuthenticationManager

  - @PostMapping("/register")
    - Accept RegisterRequest
    - Call userService.registerUser(email, password)
    - Generate access token and refresh token
    - Store refresh token hash in user record
    - Return LoginResponse (D-01: immediate access, D-02: only email/password)
    - Catch DataIntegrityViolationException and return generic error (D-04)

  - @PostMapping("/login")
    - Accept LoginRequest
    - Call authenticationManager.authenticate with UsernamePasswordAuthenticationToken
    - On success, generate tokens, store refresh hash, return LoginResponse
    - On failure, throw ApiException with INVALID_CREDENTIALS

  - @PostMapping("/logout")
    - Accept LogoutRequest
    - Validate refresh token, extract user ID
    - Clear refresh token hash in user record (invalidate)
    - Return 200 OK

  - @PostMapping("/refresh")
    - Accept RefreshTokenRequest
    - Validate refresh token (not expired, correct type)
    - Extract user ID, load user
    - Verify stored refresh token hash matches (prevents reuse after logout)
    - Generate new access token
    - Optionally rotate refresh token (generate new one, update hash)
    - Return LoginResponse with new tokens
