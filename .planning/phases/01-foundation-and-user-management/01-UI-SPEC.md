### shadcn Initialization

Run the following to initialize shadcn for this project:

```bash
npx shadcn init
```

When prompted, select:
- Style: Default
- Base color: Neutral (light theme with blue accent)
- CSS variables: Yes

Then add required components:

```bash
npx shadcn add button card input label avatar dropdown-menu dialog form toast
```

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.4 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.2 |

**Weight Consolidation:** Only 2 weights used — 400 (regular) for body text, 600 (semibold) for all emphasis (labels, headings, display).

```css

  --font-size-body: 1rem;      /* 16px */
  --font-size-label: 0.875rem; /* 14px */
  --font-size-heading: 1.25rem; /* 20px */
  --font-size-display: 1.75rem; /* 28px */
}
```

## Interaction Patterns

### Visual Hierarchy

**Focal Point:** Primary CTA button is the visual anchor on all pages.

Hierarchy order:
1. Primary CTA (Sign in, Create account, Save profile) — highest visual weight
2. Form inputs and content — secondary
3. Navigation links and secondary actions — tertiary

---

### Authentication Flow

1. **Login Page**
   - Email input (validated on blur)
   - Password input (toggle visibility)
   - "Sign in" primary button
   - "Create account" secondary link

2. **Registration Page**
   - Email input
   - Password input with strength indicator
   - Confirm password input
   - "Create account" primary button
   - "Already have an account? Sign in" link

   **Note:** Display name is NOT collected during registration per CONTEXT.md D-02. Users set display name later in profile setup.

3. **Profile Page**
   - Avatar with edit overlay on hover
   - Display name (editable)
   - Bio textarea (optional, max 500 chars)
   - Location input (optional)
   - "Save profile" primary button

### Error Handling

- Inline validation errors below each field
- Toast notifications for async errors
- Form-level error summary for accessibility

### Loading States

- Button spinner during submission
- Disabled state for interactive elements
- Skeleton loading for profile data
