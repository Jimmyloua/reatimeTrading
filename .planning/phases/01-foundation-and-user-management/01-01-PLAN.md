- D-03: Password requirements: minimum 8 characters, no complexity rules

**Entity Design Requirements:**
- User entity must have: id, email, password, displayName, avatarPath, refreshTokenHash, createdAt, updatedAt, profileComplete

- displayName can be null (for D-05, D-08)
- profileComplete defaults to false

  Create User entity and Liquibase migration following RESEARCH.md Pattern 3 exactly:

  **User.java requirements (package: com.tradingplatform.user):**
  - @Entity, @Table(name = "users")
  - @EntityListeners(AuditingEntityListener.class)
  - Lombok annotations: @Getter, @Setter, @NoArgsConstructor, @AllArgsConstructor, @Builder
  - Fields:
    - id: Long, @Id, @GeneratedValue(strategy = GenerationType.IDENTITY)
    - email: String, @Column(nullable = false, unique = true, length = 255), @Email
    - password: String, @Column(nullable = false, length = 255)
    - displayName: String, @Column(name = "display_name", length = 100), @Size(max = 100)
    - avatarPath: String, @Column(name = "avatar_path", length = 500)
    - refreshTokenHash: String, @Column(name = "refresh_token_hash", length = 64)
    - createdAt: LocalDateTime, @Column(name = "created_at", nullable = false, updatable = false), @CreatedDate
    - updatedAt: LocalDateTime, @Column(name = "updated_at"), @LastModifiedDate
    - profileComplete: boolean, @Column(name = "is_profile_complete", nullable = false), @Builder.Default false
  - Method: getDisplayNameOrFallback() returns displayName if not null, else "New User" (for D-08)
  - Method: updateProfileComplete() sets profileComplete = displayName != null && !displayName.isBlank()
