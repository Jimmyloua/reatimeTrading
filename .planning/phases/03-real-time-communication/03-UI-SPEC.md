## Spacing Scale

Inherited from Phase 1 UI-SPEC. All values must be multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

**Exceptions for Phase 3:**
- Touch targets for message send button: min 44px (accommodated via md padding + icon size)
- Chat message bubble padding: 12px vertical (not on scale, for readability)

## Interaction Patterns

### Chat Page Layout

**Route:** `/messages`

**Layout:** Two-column layout (responsive: single column on mobile)

```
+------------------+------------------------+
|                  |                        |
| Conversations    |    Chat View           |
| List             |    (selected conv)     |
| (left sidebar)   |                        |
|                  |                        |
| 280px fixed      |    1fr flexible        |
|                  |                        |
+------------------+------------------------+
```

**Mobile:** Full-screen conversation list, tap to open chat (slide-in overlay or push route).

### Conversation List Behavior

1. **Ordering:** Most recent message at top
2. **Selection:** Active conversation highlighted with accent background
3. **Unread indicator:** Red badge with count, bold text for unread conversations
4. **Presence:** Online indicator dot next to avatar
5. **Preview:** Last message snippet (truncated to 50 chars), relative timestamp

### Chat View Behavior

1. **Initial load:** Scroll to bottom, show most recent messages
2. **Auto-scroll:** New messages automatically scroll into view (if already at bottom)
3. **History load:** Scroll up to load older messages (infinite scroll)
4. **Item context:** Header shows item link, opens item detail in new tab on click
5. **Message status:** Show sent/delivered/read icons for sent messages

### Message Input Behavior

1. **Text input:** Auto-resize textarea (max 4 lines), Enter to send, Shift+Enter for newline
2. **Image upload:** Click camera icon or drag-drop, show preview before send
3. **Image constraints:** Max 5MB per image, show error if exceeded
4. **Send button:** Disabled when empty, enabled when text or image present
5. **Sending state:** Show spinner, disable input during send

### Typing Indicator Behavior

1. **Show:** When other user is typing in current conversation
2. **Hide:** 3 seconds after last typing event
3. **Position:** Below message list, above input
4. **Animation:** Fading dots or pulse animation

### Notification Behavior

1. **Real-time:** WebSocket push, update UI immediately
2. **Unread limit:** Show "+N" badge when over 50 unread
3. **Mark as read:** Click notification to mark as read and navigate to context
4. **Mark all:** Button to mark all notifications as read
5. **Retention:** Show notice if viewing notifications older than 30 days

### WebSocket Connection States

| State | UI Feedback |
|-------|-------------|
| Connected | No indicator (normal operation) |
| Connecting | Subtle spinner in header |
| Disconnected | "Connection lost. Reconnecting..." banner |
| Reconnecting | Animated reconnect attempt indicator |
| Failed | "Unable to connect. Refresh to try again." with refresh button |
