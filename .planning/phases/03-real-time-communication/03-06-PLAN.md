## Manual Verification Steps

### Prerequisites
1. Start the backend server: `cd backend && mvn spring-boot:run`
2. Start the frontend dev server: `cd frontend && npm run dev`
3. Open two browser windows (incognito for second user)

### Test 1: Chat Initiation (CHAT-01)
1. User A logs in and views a listing by User B
2. User A clicks "Contact Seller" button
3. Verify: New conversation is created and chat page opens

### Test 2: Real-time Messaging (CHAT-02)
1. User A sends a message to User B
2. Verify: Message appears instantly in User B's browser (without refresh)
3. User B replies to User A
4. Verify: Reply appears instantly in User A's browser

### Test 3: Message Persistence (CHAT-04)
1. Refresh User A's browser while viewing a conversation
2. Verify: Messages are still visible after refresh
3. Restart the backend server
4. Verify: Messages persist after server restart

### Test 4: Typing Indicator (CHAT-05)
1. User A starts typing in the chat input
2. Verify: "User A is typing..." appears in User B's chat window
3. User A stops typing for 3+ seconds
4. Verify: Typing indicator disappears

### Test 5: Notification Bell (NOTF-01)
1. User A sends a message to User B
2. Verify: User B sees notification badge increment in header
3. Verify: Notification dropdown shows the new message notification

### Test 6: Mark as Read (NOTF-04)
1. User B clicks "Mark as read" on a notification
2. Verify: Notification badge count decreases
3. Verify: Notification is marked as read in the list

### Test 7: Item Sold Notification (NOTF-02)
1. Mark an item as sold in the listings
2. Verify: Seller receives "Item Sold" notification

### Test 8: WebSocket Reconnection
1. Disconnect network (toggle offline in dev tools)
2. Verify: "Connection lost. Reconnecting..." appears
3. Reconnect network
4. Verify: Connection is restored and messages work again
