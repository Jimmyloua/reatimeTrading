| Token | Value | Usage |
|-------|-------|-------|

| lg | 24px | Section padding |

## Visual Focal Points

### Primary Screens Focal Points

| Screen | Primary Focal Point | Visual Treatment |
|--------|--------------------| --------------------|
| Transaction History List | Status badge on each transaction card | Color-coded badge positioned in top-right corner, uses status-specific color for immediate recognition |
| Transaction Detail | Timeline current step | Pulsing indicator + bold label + accent color connector line draws eye to current progress |
| Rating Form | Star rating input | Large touch targets (24px per star), amber fill color on hover/selection creates strong visual pull |
| Profile Reviews | Average rating score | 28px Display weight at top of reviews section, immediately visible |
| Dispute Form | Submit button | Destructive-styled button with warning context, draws attention to the action |

### Focal Point Hierarchy

1. **Status indicators** (badges, timeline steps) are the visual anchors because they convey the most critical information at a glance
2. **Primary action buttons** are secondary focal points with accent color background
3. **Item previews** and participant info support the focal points but do not compete for attention

## Interaction Patterns

### Transaction History Page Layout

**Route:** `/transactions`

**Layout:** Full-width with tabs

```
+------------------------------------------+
|  My Transactions                          |
+------------------------------------------+
|  [Purchases] [Sales]                      |
+------------------------------------------+
|  Filter: [All] [Active] [Completed] [...] |
+------------------------------------------+
|  TransactionCard                          |
|  TransactionCard                          |
|  TransactionCard                          |
|  ...                                      |
+------------------------------------------+
```

**Mobile:** Same layout, cards stack vertically.

### Transaction Detail Page Layout

**Route:** `/transactions/:id`

**Layout:** Two-column on desktop, single column on mobile

```
+------------------------+------------------+
| Item Preview           | Timeline         |
| (image, title, price)  | [1] Created      |
|                        | [2] Funded       |
+------------------------+ [3] Reserved    |
| Participants           | [4] Delivered    |
| Buyer: [avatar] [name] | [5] Confirmed    |
| Seller: [avatar] [name]| [6] Settled      |
+------------------------+ [7] Completed    |
| Actions                |                  |
| [Primary Action]       |                  |
| [Secondary Action]     |                  |
+------------------------+------------------+
```

### Transaction Timeline Behavior

1. **Visual Progress:** Vertical timeline with connected nodes
2. **Completed Steps:** Green checkmark icon, solid connector line
3. **Current Step:** Blue pulsing indicator, label in bold
4. **Pending Steps:** Gray outline icon, dotted connector line
5. **Timestamps:** Show date/time for completed steps, "Pending" for future
6. **Skipped Steps:** Some transactions may skip steps (e.g., direct pickup may skip DELIVERED)

### Transaction Card Behavior

1. **Click:** Navigate to transaction detail page
2. **Status Badge:** Color-coded, positioned in top-right
3. **Action Preview:** Show next required action in subtitle
4. **Unread Indicator:** Bold border or dot for new transactions requiring action
5. **Relative Time:** Show "Created X ago" or "Updated X ago"

### Star Rating Input Behavior

1. **Hover:** Preview star fill on hover
2. **Click:** Set rating, fill all stars up to clicked star
3. **Keyboard:** Arrow keys to adjust, Enter to confirm
4. **Accessibility:** Announce "X out of 5 stars selected"
5. **Default:** No selection (user must explicitly rate)

### Rating Submission Flow

1. **Post-SETTLED:** Show "Leave a Review" button on transaction detail
2. **Click:** Open rating modal or navigate to rating page
3. **Form:** Star rating (required) + review text (optional, max 500 chars)
4. **Submit:** Show "Review submitted" confirmation
5. **Pending State:** Show "Your review is pending..." message until revealed
6. **Reveal:** When both parties rate or 14-day window closes
