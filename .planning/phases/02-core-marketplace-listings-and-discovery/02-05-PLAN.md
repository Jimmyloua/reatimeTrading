<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
    Complete listing and discovery feature set:
    - Database schema for categories, listings, listing_images
    - Listing CRUD backend with ownership validation
    - Multiple image upload with primary selection
    - Full-text search with MySQL FULLTEXT
    - Dynamic filtering by category/price/condition/location
    - Frontend pages for create/edit/browse/detail
    - Image uploader with drag-and-drop
    - Category select with hierarchy
  </what-built>
  <how-to-verify>
    ## Manual Verification Checklist

    ### 1. Listing Creation (LIST-01 to LIST-05)
    1. Start backend: `cd backend && mvn spring-boot:run`
    2. Start frontend: `cd frontend && npm run dev`
    3. Login as a registered user
    4. Navigate to "Sell" or "/listings/create"
    5. Create a new listing:
       - Enter title (min 3 chars, max 200)
       - Enter description (max 5000 chars)
       - Enter price (positive number)
       - Select category from 3-level hierarchy (e.g., Electronics > Phones & Tablets > Smartphones)
       - Select condition (NEW, LIKE_NEW, GOOD, FAIR, POOR)
       - Enter location (city, region)
       - Upload 2-3 images via drag-and-drop
       - Set one image as primary
       - Submit the form
    6. Verify listing is created and redirects to detail page

    ### 2. Image Upload (LIST-02)
    1. Verify images display in correct order
    2. Verify primary image is highlighted
    3. Try uploading more than 10 images (should show error)
    4. Try uploading non-image file (should show error)

    ### 3. Listing Edit/Delete (LIST-06, LIST-07)
    1. As owner, navigate to your listing
    2. Click "Edit" button
    3. Modify title, price, condition
    4. Add/remove images
    5. Save changes
    6. Verify changes reflected
    7. Test delete functionality (soft delete)
    8. Verify listing no longer appears in browse

    ### 4. Status Management (LIST-08)
    1. Navigate to your listing
    2. Change status: AVAILABLE -> RESERVED -> SOLD
    3. Verify status badge changes color
    4. Verify only owner can change status

    ### 5. Browse and Search (DISC-01 to DISC-05)
    1. Navigate to "/listings"
    2. Test category filter (browse by category)
    3. Test text search (e.g., "laptop gaming")
    4. Test price range filter
    5. Test condition filter
    6. Test location filter (city, region)
    7. Combine multiple filters
    8. Test pagination
    9. Test sorting (newest, price low-high, price high-low)

    ### 6. Listing Detail (DISC-06, DISC-07)
    1. Click on a listing from browse page
    2. Verify all listing info displays:
       - Title, description, price
       - Condition and status badges
       - Location
       - All images in gallery
       - Seller info with profile link
    3. Click seller profile link
    4. Verify seller's other listings

    ### 7. Authorization Checks
    1. As User A, create a listing
    2. As User B, try to edit User A's listing (should fail)
    3. As User B, try to delete User A's listing (should fail)
    4. Verify 403 Forbidden responses
  </how-to-verify>
  <resume-signal>Type "approved" if all verifications pass, or describe any issues found</resume-signal>
